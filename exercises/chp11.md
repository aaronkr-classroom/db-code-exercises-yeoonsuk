1. 다음 중 물리적 스키마 모텔링의 주요 업무에 대한 설명이 잘못된 것은?
   1. 논리적 스커미를 기초로 하여 속성의 데이터 타입과 크기, 제약조건 등을 정의한다.
   2. 내부 스키마를 ER 다이어그램으로 표현한다. <--------------------------------------
   3. 시스템 성능을 고려하여 물리적 구조를 변경한다.
   4. 데이터 사용량 예측 등을 통해서 인텍스나 뷰 설계 작업을 수행한다.

2. 다음 릴레이션 스키마와 무결성 제약조건을 기초로 하여 내부 스키마를 작성하시오. 단, 각 속성의 데이터 타입과 크기는 속성의 특성을 고려해서 임의로 결정하시오.

> 도서(울륜벌효/ 제목/ 저자/ 출판사/ 발행일， ISBN , 정가)


| No. | 무결성제약조건 | 제약유형 | 관련랄레이션 |
|-----|---------------|---------|-------------|
| 1 | 제목, 저자, ISBN은 널 값을 가질 수 없다. | NOT NULL | 도서 |
| 2 | ISBN은 유일한 값을 가져야 한다. | UNIQUE | 도서 |

CREATE TABLE 도서 (
  분류번호 CHAR(10) PRIMARY KEY,
  제목 VARCHAR(100) NOT NULL,
  저자 VARCHAR(50) NOT NULL,
  출판사 VARCHAR(50),
  발행일 DATE,
  ISBN CHAR(13) NOT NULL UNIQUE,
  정가 INT
);

3. 성능 향상을 위한 물리적 구조 변경 방법 가운데 테이블 추가에 대한 설명이 잘못된 것은?
   1. 지주 사용되는 통계 데이터가 여러 테이블에 산재된 데이터를 기초로 하여 생성되는 경우, 별도의 통계 테이블을생성할 수 있다.
   2. 대용량 테이블의 특정 부분만 주로 시용되는 경우 해당 부분만 분리해서 별도의 테이블을 생성할 수 있다. <----------------------------
   3. 대용량 테이블의 특정 부분만 주로 사용되는 경우 해당 부분만 복사해서 별도의 태이블을 생성할 수 있다.
   4. 하나의 업무 단위가 시간의 흐름에 따라 반복적으로 발생하거나 변경되는 이력 또는 진행 이력이 필요한 경우， 이력 태이블을 추가할 수 있다.

4. 성능 향상을 위한물리적 구조 변경 방법 가운데 테이블 분할에 대한 설명이 잘못된 것은?
   1. 한 테이블 안에서 검색 위주의 칼럼과 갱신 위주의 칼럼이 명확히 구분되는 경우, 각각을 별도의 테이블로 분할해서 관리한다.
   2. 한 테이블 안에서 특정 칼럼의 보안이 필요한 경우 해당 칼럼만분할해서 관리한다.
   3. 한태이블안에서 특정 칼럼의 크기가매우큰경우 이 칼럼만분할해서 관리한다.
   4. 한테이블 안에서 특별히 빈번하게 갱신되는 칼럼이 있는 경우， 이들 칼럼만분할해서 관리한다. <-------------------------------------

5. 성능 향상을 위한 물리적 구조 변경 방법 가운데 칼럼 중복에 대한 설명이 잘못된 것은?
   1. 자주 검색되는 칼럼이 자주 변경되는 경우 해당 칼럼을 중복시킬 수 있다. <------------------------------------------------------
   2. 다른 칼럼 값을 기초로 하여 계산된 합계， 평균， 개수 등의 파생된 값이 많이 활용되는 경우, 파생 칼럼을 추가할 수 있다.
   3. 기본키가 여러 가지 정보를 포함하고 있는데 그 일부가 지주 사용되는 경우， 해당 정보를 분리해서 별도의 칼럼으로 추가할 수 있다.
   4. 자주 검색되는 칼럼에 변경이 거의 일어나지 않는 경우 해당 칼럼을 중복시킬 수 있다.

6. 인텍스의 특정이 아닌 것은?
   1. 인텍스는 DBMS에 의해 자동으로 시용되고 유지된다
   2. 인텍스는 테이블과는 논리적 물리적으로 독립되어 있다.
   3. 기본 태이블을 제거해도 인텍스는 유지된다. <----------------------------------------
   4. 인텍스를 너무 많이 생성하면 오히려 데이터 조작어의 처리 효율이 저하된다.

7. 인텍스를 선정할 때의 고려 사항을 잘못 설명한 것은?
   1. 인텍스가 많으면 갱신 처리를 할 때 비효율적이다.
   2. 테이블의 데이터양이 적은 경우 인텍스를 생성하지 않는 것이 더 효율적이다.
   3. 검색 효율이 가장 중요하므로 가능하면 많은 인텍스를 생성히논 것이 좋다. <------------------------------------------
   4. 검색 대상이 넓은 경우 인텍스를 시용하는 것은 오히려 비효율적이다.

8. 다음 제품 릴레이션을 테이블로 표현했을 때， 브랜드명 칼럼과 입고일 칼럼에 각각 인텍스를 생성하는 것이 적절한지 여부를 설명하시오. 단， [참고 사항]을 고려한다.


> 제품(젠풀」초의/ 제품명/ 브랜드명/ 원가/ 판매가/ 수량/ 입고일/ 담당자 id(FK))

**참고사항**

   1. 제품 테이블에 저장된 데이터가 5,000 개다
   2. 브랜드명의 종류는 200 개다
   3. 제품을 검색할 때 주로 브랜드명을 이용해서 검색한다.
   4. 검색 결과를 입고일 기준으로 내림치순으로 출력하는 경우가 많다
   5. 모든 제품은 매주 월요일에 입고된다
   6. 제품 테이블에 저장된 데이터는 2024년 10월 7일부터 2024년 10 월 28일까지 입고된 제품 데이터다

- 브랜드명 인덱스: 적절함
브랜드 종류가 200개이고, 제품 검색 시 주로 브랜드명을 사용하므로 검색 효율 향상에 도움이 됩니다.

- 입고일 단독 인덱스: 적절하지 않음
모든 제품이 월요일에만 입고되어 입고일 값의 종류가 적고, 주로 정렬 기준으로 쓰이므로 단독 인덱스 효과가 크지 않습니다.
필요하다면 브랜드명 + 입고일 복합 인덱스가 더 적절합니다.

11. 다음과 같은 뷰 정의서를 기초로 하여 고객등급-뷰를 생성하는 SQL 명령문을 작성하시오.

> 고객(곤표 id ， 이름/ 휴대폰/ 주소/ 직장명/ 누적구매액/ 방문일수/ 현재등급)

| 뷰이름 | 용도 | 기보 테이블 | 컬럼 | 데이터 타입 |
|-------|------|------------|------|------------|
| 고객등급_뷰 | 고객의 신규등급판정 프로그램에서사용함 | 고객 | id | VARCHAR2(20) |
| 고객등급_뷰 | 고객의 신규등급판정 프로그램에서사용함 | 고객 | 고객명 | VARCHAR2(20) |
| 고객등급_뷰 | 고객의 신규등급판정 프로그램에서사용함 | 고객 | 구매액 | NUMBER(10) |
| 고객등급_뷰 | 고객의 신규등급판정 프로그램에서사용함 | 고객 | 방문일수 | NUMBER(3) |
| 고객등급_뷰 | 고객의 신규등급판정 프로그램에서사용함 | 고객 | 등급 | CHAR(5) |

CREATE VIEW 고객등급_뷰 AS
SELECT
  고객_id AS id,
  이름 AS 고객명,
  누적구매액 AS 구매액,
  방문일수,
  현재등급 AS 등급
FROM 고객;
