- 고객(Customer)
- 관리자(Admin)

[요구사항]
1. 고객은 상품 목록을 조회할 수 있어야 한다.
2. 고객은 상품의 이름, 카테고리, 가격, 재고 정보를 확인할 수 있어야 한다.
3. 고객은 원하는 상품을 선택하여 주문할 수 있어야 한다.
4. 주문 정보에는 고객 정보, 상품 정보, 수량, 주문 날짜가 저장되어야 한다.
5. 관리자는 상품 정보를 등록하고 관리할 수 있어야 한다.
6. 관리자는 전체 주문 내역을 조회할 수 있어야 한다.
7. 관리자는 상품 정보를 정렬하여 조회할 수 있어야 한다.
8. 사용자는 조건에 맞는 상품을 검색할 수 있어야 한다.

# 온라인 쇼핑몰 DB 설계

## 1. 주제
온라인 쇼핑몰

## 2. 엔티티
- Customer
- Product
- Orders

## 3. 엔티티 설명
Customer는 쇼핑몰 고객 정보를 저장하는 테이블이다.  
Product는 판매 상품 정보를 저장하는 테이블이다.  
Orders는 고객이 어떤 상품을 몇 개 주문했는지를 저장하는 테이블이다.

## 4. 관계
- Customer 1 : N Orders
- Product 1 : N Orders

즉, 한 명의 고객은 여러 번 주문할 수 있고, 하나의 상품도 여러 주문에 포함될 수 있다.

## 5. 구현 내용
- CREATE TABLE로 3개의 테이블 생성
- 각 테이블에 5개 이상의 데이터 입력
- 전체 조회(SELECT)
- 정렬(ORDER BY)
- 조건 검색(WHERE)
- JOIN을 사용한 주문 조회 작성
